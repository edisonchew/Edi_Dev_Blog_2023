Week 6 (Part 1) - JavaScript Basics Recap

What I have learnt:

  • Definition of variable:

    • A variable is a named container for a value (Used to store, manipulate & retrieve data)
    • For e.g. let quantity = 3;
      console.log(quantity)
    • The name of the variable can be anything so long as it is meaningful.
    • Each variable must have a Unique name

  • Statements

    • console.log('Welcome!');

  • Data Types (Strings, Values, Booleans)

    • 3 Primitive Data Types
      • Numbers: Floating point numbers, decimals & Integers
      • Strings: Sequence of characters, text
      • Booleans: Logical Data (True or False)
    • 2 Other Primitive Data Types
      • Undefined: Data type of a variable that does not have a value yet
      • Null: Also means non-/nothing
    • Numbers = 0.75 (NO QUOTES)
    • Strings = "Hi, Bryan!" (ENCLOSED IN QUOTES WHICH CAN BE SINGLE OR DOUBLE QUOTES, BUT MUST MATCH)
    • Booleans = true (EITHER TRUE OR FALSE)
    • Examples:
      • Numbers: let x = 1
      • Strings: let myName = "Bob";
      • Boolean: let myName = true;
      • Array: let myName = [1, 2, 3, "hi"];

  • Template Literals

    • Template literals are enclosed by the back-tick (``)
    • Example:
      • Usual method of writing strings:
        var age = 21;
        console.log('My age is ' + age);
      • Substitution using Template String:
        var age = 21;
        console.log(`My age is ${age}`);

  • Comparison & Logical Operators

    • Strict Equal To (===):
      • This operator compares two values to check both the data type and value are the same
      • Example:
        • '8' === 8 returns False;
          Because they are no the same data type or value
        • '8' === '8' returns True;
          Because they are the same data type and value
    • Strict Not Equal To (!==)
      • This operator compares two values to check that both the data type and value are not the same
        • '8' !== 8 returns True;
          Because they are not the same data type or value
        • '8' !== '8' returns False;
          Because they are the same data type and value
    • Is Equal To (==)
      • This operator compares two values (nunmbers, strings, or Booleans) to see if they are the same
        • 'Hello' == 'Goodbye' returns False;
          Because they are not the same value
        • 5 == '5' returns True;
          Because they have the same value althought their data types are different
    • Is Not Equal To (!=)
      • This operator compares two values (numbers, strings, or Booleans) to see if they are not the same
        • 'Hello' != 'Goodbye' returns True;
          Because they are not the same string
        • 'Hello' != 'Hello' returns False;
          Because they are the same string
    • Greater than (>)
      • This operator checks if the number on the left is greater than the number on the right
        • 8 > 7 returns True
        • 7 > 8 returns False
    • Lesser than (<)
      • This operator checks if the number on the left is lesser than the number on the right
        • 7 < 8 returns True
        • 8 < 7 returns False
    • Greater Than Or Equal To (>=)
      • This operator checks if the number on the left is greater than or equal to the number on the right
        • 8 >= 7 returns True
        • 7 >= 8 returns False
        • 8 >= 8 returns True
    • Lesser Than Or Equal To (<=)
      • This operator checks if the number on the left is lesser than or equal to the number on the right
        • 8 <= 7 returns False
        • 7 <= 8 returns True
        • 8 <= 8 returns True
    • Examples of using the Comparison Operator:
      • let pass = 50;
        let score = 85;
        if (score >= pass) {
        console.log("You passed!");
        }
    • Casting (Best Practise)
      • Use == if your want to test whether the two things being compared are equal;
        Example: console.log(99 == "99");
      • Use === if you want to test whether the two things being compared are of the same type and value;
        Example: console.log(99 === "99");

  • Logical Operators

    • Logical Operators allow you to compare results of more than one comparison operator
    • Logical And (&&)
      • This operator test more than one condition
      • Example:
        ((2 < 5) && (3 >= 2))
        This condition returns True. If both expressions evaluate to true then the expression returns true. If just one of these returns false, then the expression returns false
        • true && true returns True
        • true && false returns False
        • false && true returns False
        • false && false returns False
    • Logical Or (||)
      • This operator test at least one condition
      • Example:
        ((2 < 5) || (3 >= 2))
        This condition returns True. If both expression evaluate to true then the expression returns true. If just one of these returns false, then the expression returns false
        • true && true returns True
        • true && false returns True
        • false && true returns True
        • false && false returns False
    • Not (!)
      • This operator inverts True or False values
      • Example:
        (!true)
        This condition returns False. Operator simply inverts the value contained within

  • Loops

    • Repetition aka. Loop control structure
    • Loops check a condition
    • If it returns True, a code block will run
    • Then the conditiono will be checked again and if it still remains True, the code block will run again
    • It repeats until the condition returns False.
    • Each time we perform a loop action, it is known as an iteration.
    • Loop, provide a way to repeat the same set of actions over and over again

  • For Loops

    • Example:
      for (let i = 0; i < 10; i++) {
      console.log("Practice:" + 1);
      }
    • Output: Practice: 0 to Practice 9; Loop stops because i < 10
    • INITIALISATION:
      • Create a variable and set it to 0, acts as a counter
        let i = 0;
        Variable is only created the first time the loop is run
    • CONDITION:
      • Loop continues to run until the counter reaches a specific number
        i < 10;
        The value of i was initally set to 0, so in this case the loop will run 10 times beofre stopping
    • INCREMENT/UPDATE:
      • Every time the loop has run the statements in the curly braces, it adds one to the counter
        i++
        One is added to the counter using the increment (++) operator, it is also possible for loops to count downwards using the decrement operator (--)

  • While Loops

    • Example:
      let i = 0;
      while (i < 10) {
      console.log("Practice:" + 1);
      i++;
      }
    • Output: Practice 0 to Practice 9; Loop stops beacuse i < 10
    • Basically a while loop is the same as a for loop; however, sometimes, you may not know how many times it will iterate (repeat)
    • The while loop repeats as long as the condition is True
    • One disadvantage of using while loop is that it's easier to end up with an infinite loop, which may cause your computer to hang because the loop will forever until the browser is out of memory

  • Functions

    • Functions let you group a series of statments together to perfrom a specific task
    • Functions are reusable and save you from writing out the same code over and over
    • Declaring A Function:
      function sayHello() {
      document.write('Hello');
      }
      sayHello();
    • Declariing A Function That Needs Information:
      function getArea (width, height) {
      return width * height;
      }
      getArea(3, 5);

Week 6 (Part 2) - JavaScript Objects & More

What I have learnt:

  • Object Oriented Programming (OOP)

    • Objects group together a set of variables and functions to create a model
    • Literal Objects
      • let hotel = {
        name: 'Raffles Hotel', (PROPERTIES)
        rooms: 100, (PROPERTIES)
        booked: 24, (PROPERTIES)
        gym: true, (PROPERTIES)
        roomTypes: ['twin', 'suite', 'deluxe'], (PROPERTIES)
        checkAvailability: function() { (METHOD)
        return this.rooms - this.booked; (METHOD)
        }
        };
    • Accessing An Object
      • let hotelName = hotel.name
        let roomsFree = hotel.checkAvailability();
        let hotelName = hotel['name'];
        let roomsFree = hotel['checkAvilability']();
    • Difference between Literal Object & Function-Based Objects
      • Literal Objects:
        • Has no constructors
        • Cannot be inherited
        • Best used for "one off" objects where only one copy will exist
        • A colon separates the key/value pair
        • For global or configuration objects like 'game settings'
      • Function-Based Objects
        • Has a constructor
        • Can be inherited (but very messy, out of scopre here)
        • Best use for stuff that you want to instantiate again and again
        • The this keyword is used instead of the object name
        • If you have lots of objects with similar functionality
    • VAR, LET, CONST
      • VAR:
        • Variable declarations using VAR are treated as if they are at the top of the function (or global scope, if declared outside of a function) regardless of where the actual declaration occurs; this is called hoisting.
        • The scope of a variable defined with VAR is function scope or global outside any function, global.
        • VAR allows us to re-declare a variable
        • Example:
          var course = 'IMGD';
          var course = 'A3DA'
      • LET:
        • let declarations are not hoisted to the top of the enclosing block
        • let is block-scoped meaning it only exist within the braces it is contained in {...}
        • LET does not allow us to re-declare a variable
        • Example:
          let course = 'IMGD'
          let course = 'A3DA'
          Uncaught SyntaxError: Identifier ' course' has already been declared
      • CONST
        • Variables declared using const are considered constants, meaning their values cannot be changed once set
        • For this reason, everyy const variable must be initialized on declaration
