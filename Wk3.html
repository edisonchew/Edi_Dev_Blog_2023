Week 3 - Forms & Advance CSS Layouts

What I have learnt:

Tags:

  • Form tag(forms) --> Element used to create an HTML form for user input (For different types of input elements, such as: text fields, checkboxes, radio buttons, submit buttons, etc.)
    • action --> the URL to send form data to
    • method --> the type of HTTP request
  • Input tag (input) --> Element is the most used form element (Can be displayed in many ways, depending on the type attribute
    • input text = "text" --> Displays a single-line text input field
    • input text = "radio" --> Displays a radio button (for selecting one of many choices)
    • input text = "checkbox" --> Displays a checkbox (for selecting zero or more of many choices)
    • input text = "submit" --> Displays a submit button (for submitting the form)
    • input text = "button" --> Displays a clickable button
  • Label tag (label)
    • Defines a label for many form elements
    • Element is useful for screen-reader users, because the screen-reader will read out loud the label when the user focuses on the input element
    • Element also helps users who have difficulty clicking on very small regions (such as radio buttons or checkboxes) - because when the user clicks the text within the label element, it toggles the radio button/checkbox.
    • The for attribute of the label tag should be equal to the id attribute of the input tag element to bind them together
  • Media Queries --> Used to modify the CSS of a site depending on qualities such as viewport size or device type
  • Positioning --> CSS position property lets you alter the positioning scheme of a particular element
    • Relative Position
      • Moves elements around relative to where they would appear on the page.
      • Uses nudging boxes around
      • Think of the offsets as being applied after the browser finishes laying out the page. (Uses Top, Left, Right, Bottom properties)
    • Absolute Position
      • Absolute position elements are placed in relation to a parent content(browser viewport, or parent element).
      • Positions can be modified using positioning offsets, top, right, bottom, left
      • Offset is relative to the entire browser window instead of the original position of the element
    • Fixed Position
      • Very similar to absolute EXCEPT the coordinate system is relative to the entire browser window.
      • Fixed elements dont scroll with the rest of the page
      • Commonly known as sticky element
    • Z-index
      • z-index property lets you control the depth of elements on the page
      • Negative z-index go further into the page (behind)
      • Positive z-index comes out of the page
      • An element with a higher z-index value overlaps an element with a lower z-index value
      • Positioned elements have a z-index of 0 by default
      • z-index ONLY on elements with a position property set to absolute, fixed, or relative. (if you set a z-index to an element with no position, it will do nothing)
    • FlexBox
    • Wireframes
