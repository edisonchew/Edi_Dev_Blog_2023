Week 1 - Introduction to Web Development

HTML (Hypertext Markup Language)

--> HTML is a standard markup language for creating web pages and web applications.

What I have learnt:

Tags:

  • Image Tag (img) --> Used to display an image
  • Unordered list (ul) --> Using bullets points to list items
  • Ordered list (ol) --> Using number to list items
  • Paragraph (p) --> Defines a paragraph
  • Headings (h1-h6) --> Defines HTML headings (h1 -> Most important heading, h6 -> Least important heading)
  • break (br) --> Inserts a single line break
  • Horizontal rule (hr) --> Used to seperate content (or define a change) in a HTML page.
  • emphasis/stress text (em) --> Defines emphasized text
  • (strong) --> Defines important text by bolding the text
  • Inline frame (iframe) --> Used to embed another document within the current HTML document
  • Anchor (a) --> Defines a hyperlink
  • List (li) --> Defines a list item
  • Division (div) --> Divides the HTML documents into sections (A container unit)
  • Span (span) --> Used to mark up small chunks of HTML inside a line (An inline unti)

--> Tags in HTML is used to describe a website structure and tells the browser how to format content.

Week 2 - Semantics & CSS

Semantics

  • Relates to syntax
  • The meaning behind words, phrases. (In our case it is the HTML)

What I have learnt:

Tags:

  • Header (header) --> Used for navigation or introductory content (Can be used at the top of the page content to encapsulate navigation, logos
  • Article (article) --> Commonly used for news content, blog posts or independently content piece
  • Aside (aside) --> For content that is indirectly related to your main content (Commonly seen in sidebars or call out boxes)
  • Style (style) --> Defines style information for a document
  • Link (link) --> Defines the relationship between a document and an external resource (most used to link to style sheets)

ID Attribute - TARGETING AN INDIVIDUAL:

  • #header{} selects any element with the id "header", (p class = "header" /p)
  • Unique to that element
  • Hash symbol
  • The "#" is how you tell CSS "this is an id".

Class Attribute - TARGETING A GROUP:

  • .warning { color: red; } Selects any element with the class name "warning", e.g. (p class = "warning" /p)
  • Used to style an element grouping
  • Targets one or more element's class attribute
  • Elements can be of different type
  • An element can have only 1 id but multiple classes, so you can take advantage of that for greater flexibility

Fonts:

  • font-size --> It is a property specifies the size of a font (Can be specified as a fixed size in various units, a percentage, or as predefined keyword
    • font-size: 1.5em;
    • font-size: 12px;
    • font-size: 100%
    • font-size: larger;
  • "em' unit:
    • Lets you set the size of the text relative to the text around it.
    • Makes the page resize nicely proportion if the user changes their default font-size.
    • The default size is "1em" or 16px
    • e.g. p { font-size: 0.9em; }, strong { font-size: 1.5em; }
  • "px" unit:
    • Lets you size font in terms of pixels, which is the unit also used to size images and other elements
    • Easier to understand than em
    • Dont work as well when printing or resizing
    • e.g. h2 { font-size: 17px }
  • font-style --> Specifies the font style of the text, either "normal" by default or "italic"
  • font-weight --> Specifies the thickness of the font (Default = "normal"; Typical = "bold"; can be "bolder, "lighter", or number from 100 to 900)

Week 3 - Forms & Advance CSS Layouts

What I have learnt:

Tags:

  • Form tag(forms) --> Element used to create an HTML form for user input (For different types of input elements, such as: text fields, checkboxes, radio buttons, submit buttons, etc.)
    • action --> the URL to send form data to
    • method --> the type of HTTP request
  • Input tag (input) --> Element is the most used form element (Can be displayed in many ways, depending on the type attribute
    • input text = "text" --> Displays a single-line text input field
    • input text = "radio" --> Displays a radio button (for selecting one of many choices)
    • input text = "checkbox" --> Displays a checkbox (for selecting zero or more of many choices)
    • input text = "submit" --> Displays a submit button (for submitting the form)
    • input text = "button" --> Displays a clickable button
  • Label tag (label)
    • Defines a label for many form elements
    • Element is useful for screen-reader users, because the screen-reader will read out loud the label when the user focuses on the input element
    • Element also helps users who have difficulty clicking on very small regions (such as radio buttons or checkboxes) - because when the user clicks the text within the label element, it toggles the radio button/checkbox.
    • The for attribute of the label tag should be equal to the id attribute of the input tag element to bind them together
  • Media Queries --> Used to modify the CSS of a site depending on qualities such as viewport size or device type
  • Positioning --> CSS position property lets you alter the positioning scheme of a particular element
    • Relative Position
      • Moves elements around relative to where they would appear on the page.
      • Uses nudging boxes around
      • Think of the offsets as being applied after the browser finishes laying out the page. (Uses Top, Left, Right, Bottom properties)
    • Absolute Position
      • Absolute position elements are placed in relation to a parent content(browser viewport, or parent element).
      • Positions can be modified using positioning offsets, top, right, bottom, left
      • Offset is relative to the entire browser window instead of the original position of the element
    • Fixed Position
      • Very similar to absolute EXCEPT the coordinate system is relative to the entire browser window.
      • Fixed elements dont scroll with the rest of the page
      • Commonly known as sticky element
    • Z-index
      • z-index property lets you control the depth of elements on the page
      • Negative z-index go further into the page (behind)
      • Positive z-index comes out of the page
      • An element with a higher z-index value overlaps an element with a lower z-index value
      • Positioned elements have a z-index of 0 by default
      • z-index ONLY on elements with a position property set to absolute, fixed, or relative. (if you set a z-index to an element with no position, it will do nothing)
    • FlexBox
    • Wireframes

Week 4 - Putting it All Together

What I have learnt:
  • Why is Responsive Images needed?
    • Auto-resize based on screen size
    • Better performance
      • Load higher resolution images only when needed
    • Provide better design control depending on device
  • How to use Responsive Images?
    • To scale up and down: .responsive-img { width: 100%; height: auto;}
    • When images to scale never larger than original size: .responsive-img-max {max-width: 100%; height: auto;} / .responsive-img-max-fixed-width { max-width: 100%; width: 400px; height: auto;
      • The max-width property sets a maximum width for an element, which does not allow the width of that element to be larger than its max-width value (but it can be smaller).
    • For different resolutions: Use srcset
    • To fit images to box: Auto fit to box (.responsive-img-cover { background-image: url (...); background-size: cover;})
